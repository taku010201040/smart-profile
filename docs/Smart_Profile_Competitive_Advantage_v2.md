## 1. エグゼクティブサマリー

Smart Profileは「日常の記録をAIが自動分析し、プロフィールを生成する」学生特化型のライフログ＆マッチングプラットフォームである。本レポートでは、初回調査の10サービス（YOUTRUST、Notion、LinkedIn、Wantedly、Yenta、Humane AI Pin、Penmark、Mem.ai / Rewind AI、Pairs、LAPRAS）に加え、追加調査の8サービス（**Bee、Omi、Plaud NotePin、Limitless、Apple Journal、Day One、Otter、Reflect**）を含む計18の競合サービスを分析した。

分析の結果、Smart Profileが狙うべき**市場の空白地帯**が明確になった。それは、「全部を自動で録る」「価値観や人生の変化まで長期で整理する」「学生の進路や就活の意思決定支援に落とす」の3つを同時に実現するポジションである。現時点で、この3要素を兼ね備えたサービスは市場に存在しない。

## 2. 競合の全体マップ

### 2.1. 4象限マッピング

競合サービスは、その機能特性に基づいて以下の4つの象限に分類できる。

| 象限 | 特徴 | 該当サービス |
|---|---|---|
| **常時・半常時の記録** | 会話や行動を自動的に記録し続ける | Bee、Omi、Limitless（販売終了） |
| **録音・文字起こし・議事メモ** | 特定の場面（会議・授業）の記録に特化 | Plaud NotePin、Otter |
| **自己理解・内省** | 日記や振り返りを通じた自己理解を支援 | Apple Journal、Day One |
| **個人ナレッジベース化** | 情報を整理・検索可能な知識として蓄積 | Mem.ai、Reflect、Notion |

これに加え、前回調査のキャリアSNS・マッチング領域（YOUTRUST、LinkedIn、Wantedly、Yenta、Pairs）と、学生特化領域（Penmark）、エンジニア特化領域（LAPRAS）が存在する。

Smart Profileは、**これらの象限を横断する唯一のサービス**として位置づけられる。記録の自動化（象限1）、自己理解の深化（象限3）、ナレッジの蓄積（象限4）を統合し、さらにプロフィール生成とマッチングという独自の出口を持つ。

## 3. 直接競合との詳細比較

### 3.1. vs. Bee — 最も近い直接競合（脅威度：★★★★★）

Beeは$49.99のウェアラブルAIデバイスで、「会話・タスク・場所を要約し、個人インサイトやリマインダーに変える」パーソナルAIを掲げている [1]。2025年7月にAmazonに買収され、Amazonのエコシステムに組み込まれつつある [2]。バックグラウンドでパターン・好み・人間関係を学習する方向性は、Smart Profileの思想に最も近い。

| 比較項目 | Smart Profile | Bee |
|---|---|---|
| **形態** | Webアプリ（無料） | ウェアラブルデバイス（$49.99）+ アプリ |
| **データ収集方法** | テキスト・音声・画像を能動的に記録 | 常時装着で会話を自動記録 |
| **AI分析の方向性** | スキル・興味・強みを抽出し**プロフィール化** | パターン・好みを学習し**リマインダー・インサイト化** |
| **データの出口** | プロフィール生成 → マッチング → 名刺 | 要約・リマインダー・検索（自己完結） |
| **ターゲット** | 学生（キャリア形成・自己理解） | 一般消費者（記憶補助・生産性向上） |
| **プライバシー** | ユーザーが記録タイミングを制御 | 常時録音（プライバシー懸念大） |

**Smart Profileの優位性:** Beeは「記録して思い出す」ことに特化しているが、蓄積したデータを**対外的に共有可能なプロフィールに変換する機能を持たない**。Beeのインサイトは個人の中で閉じるが、Smart Profileは記録データを「プロフィール→マッチング→名刺」という形で外部に展開する。さらに、Beeはハードウェア購入が必要だが、Smart Profileはスマートフォンのみで完結する。学生にとって$49.99のデバイス購入はハードルが高く、Webアプリとして無料で始められるSmart Profileの方がアクセシビリティが格段に高い。

また、Beeは「価値観の変化」や「人生の岐点での意思決定支援」を明示的に提供していない。Smart Profileが「過去の自分との比較」や「成長の可視化」を打ち出すことで、Beeとの差別化を明確にできる。

### 3.2. vs. Omi — ウェアラブル×メモリーの競合（脅威度：★★★★☆）

Omiは$89のウェアラブルAI（ペンダント型）で、会話をメモリー・タスク・リマインダーに変換する [3]。CES 2025では脳波読み取り機能も発表しており、技術的な野心は高い。オープンソースプラットフォームとしてコミュニティ主導の開発を進めている点も特徴的である。

**Smart Profileの優位性:** OmiもBeeと同様、データの出口が「個人の記憶補助」に留まっている。会話からタスクやリマインダーを生成する機能はあるが、**蓄積データからプロフィールを自動生成し、他者とのマッチングに活用する**という発想は持っていない。また、Omiのターゲットはテック好きのビジネスパーソンであり、学生向けの機能設計（自己分析、就活支援、友人マッチング）は一切ない。Smart Profileは「学生の成長を可視化し、キャリア形成を支援する」という明確なユースケースを持つ点で差別化される。

### 3.3. vs. Plaud NotePin / Otter — 録音・文字起こし特化型（脅威度：★★★☆☆）

Plaud NotePinは$179のウェアラブル型AIノートテイカーで、150万人以上のユーザーを持つ [4]。ISO 27001、GDPR、SOC 2、HIPAA対応など、セキュリティとコンプライアンスを強く押し出している。Otterはリアルタイム文字起こし・要約・アクションアイテム抽出に特化したAIミーティングアシスタントである [5]。

**Smart Profileの優位性:** 両サービスとも「特定の場面（会議・授業・面談）の記録と整理」に特化しており、日常生活全体のライフログではない。録音データは議事録や要約として消費されるが、**長期的な蓄積データからユーザーの成長やスキルを抽出する機能はない**。学生が授業メモとしてPlaudやOtterを使う可能性はあるが、Smart Profileは「授業の記録」ではなく「学生生活全体の記録から自己理解を深める」という上位の価値を提供する。両者は代替関係ではなく、むしろSmart Profileの記録手段の一つとして補完的に位置づけられる。

### 3.4. vs. Apple Journal / Day One — 自己理解・内省型（脅威度：★★★☆☆）

Apple JournalはiPhoneネイティブの無料ジャーナルアプリで、写真・動画・音声・場所・気分をもとに記録できる [6]。オンデバイスAI処理によるプライバシー保護と、提案機能による習慣化の設計が強い。Day OneはAIによる深掘り質問・要約・タイトル提案を行うジャーナルアプリで、オプトイン方式のプライバシー設計が特徴である [7]。

**Smart Profileの優位性:** 両サービスは「自分を振り返って自己理解を深める」体験では強力だが、**記録が個人の日記として閉じている**。Apple JournalもDay Oneも、蓄積した記録から対外的に共有可能なプロフィールを生成したり、他者とマッチングしたりする機能は一切持たない。Smart Profileは「内省」を「行動」に変換する点で差別化される。つまり、自己理解で終わるのではなく、理解した自分を**プロフィールとして可視化し、適切な人や機会とマッチングする**という次のステップを提供する。

ただし、常時録音を必要としないApple JournalやDay Oneの方が、プライバシー面でユーザーの心理的ハードルが低い点は認識すべきである。Smart Profileも「記録するタイミングはユーザーが制御する」設計であるため、この点では同等のプライバシー保護を提供できる。

## 4. 間接競合との差別化

### 4.1. vs. Wantedly — AIプロフィール自動生成の直接競合（脅威度：★★★★☆）

Wantedlyは2025年12月に「プロフィール自動生成」機能をリリースした [8]。履歴書・職務経歴書のPDFをアップロードすると、AIが情報を抽出してプロフィールを整形する。学生向け機能（Wantedly for students）も展開しており、自分史ベースのキャリア軸発見を支援している。

**Smart Profileの優位性:** Wantedlyのアプローチは「既存の書類を整形する」静的なものであり、Smart Profileの「日常記録から新しい気づきを発見する」動的なアプローチとは根本的に異なる。学生は職務経歴書を持っていないことが多く、Wantedlyの自動生成では空のプロフィールしか作れない。Smart Profileは「まだ言語化されていない経験や強み」をAIが発掘する点で、学生にとってはるかに実用的である。

### 4.2. vs. LAPRAS — エンジニア特化 vs. 全学生対象（脅威度：★★☆☆☆）

LAPRASはGitHub・Qiitaなどの公開情報を自動収集し、エンジニアのポートフォリオを自動生成する [9]。148万人以上のプロフィールを保有し、機械学習によるスコアリングで技術力と市場価値を可視化する。

**Smart Profileの優位性:** LAPRASは「エンジニアの公開アウトプット」に限定されるが、Smart Profileはあらゆる学生の日常活動をデータソースとする。文系・体育会系・芸術系など、GitHubにアウトプットを持たない大多数の学生にも対応できる点で、市場規模が格段に大きい。

### 4.3. vs. YOUTRUST / LinkedIn — キャリアSNS（脅威度：★★☆☆☆）

YOUTRUSTは信頼ベースのキャリアSNS [10]、LinkedInはグローバルなプロフェッショナルSNSである。両サービスともユーザーが手動でプロフィールを作成・更新する必要があり、「自分の経験が眠っていて説明できない」という学生の課題を解決しない。また、両サービスとも社会人中心であり、学生のコミュニティ形成には適していない。

### 4.4. vs. Penmark — 学生向けサービス（脅威度：★★☆☆☆）

Penmarkは大学生向けの時間割アプリで、シラバスからの時間割作成、授業ごとのトークルーム、学内掲示板を提供する [11]。学生特化という点では共通するが、Penmarkは「履修管理」に特化しており、自己理解やキャリア形成の支援機能は持たない。両者は補完関係にあり、将来的な連携も視野に入る。

### 4.5. vs. Yenta / Pairs — マッチング（脅威度：★☆☆☆☆）

Yentaは完全審査制のAIビジネスマッチング [12]、Pairsは恋愛マッチングアプリである。Smart Profileは友人・恋人・ロールモデル・企業という4カテゴリのマッチングを提供し、AIが分析した日常記録に基づく「価値観ベースの深いマッチング」を実現する点で差別化される。

## 5. Smart Profileの決定的優位性

### 優位性1：「記録するだけ」でプロフィールが育つ — 唯一の動的プロフィール生成

既存サービスの全てが、プロフィール作成においてユーザーに何らかの能動的作業を求める。LinkedInもWantedlyもYOUTRUSTも手動入力が必要であり、LAPRASはエンジニアの公開アウトプットに限定される。Wantedlyの自動生成も「既存書類の整形」に過ぎない。

一方、Bee・Omi・Limitlessのようなライフログ系サービスは記録の自動化には成功しているが、**蓄積データをプロフィールに変換する機能を持たない**。記録は個人の中で閉じ、対外的な共有手段がない。

Smart Profileは、この両者の間に存在する**「記録の自動蓄積」と「プロフィールの自動生成」を接続する唯一のサービス**である。日常の記録を蓄積するだけで、AIがスキル・興味・強みを自動抽出し、プロフィールを生成・更新し続ける。

### 優位性2：データの「出口」がある — 記録→分析→プロフィール→マッチングの一気通貫

競合サービスのデータ活用範囲を比較すると、Smart Profileの独自性が際立つ。

| サービス | データの入口 | データの出口 |
|---|---|---|
| **Bee** | 会話の自動記録 | 要約・リマインダー（自己完結） |
| **Omi** | 会話の自動記録 | タスク・メモリー（自己完結） |
| **Plaud / Otter** | 会議・授業の録音 | 議事録・要約（自己完結） |
| **Apple Journal / Day One** | 手動の日記記録 | 振り返り・内省（自己完結） |
| **Mem.ai / Reflect** | メモ・音声の記録 | 検索可能なナレッジベース（自己完結） |
| **Wantedly** | PDF書類のアップロード | 整形されたプロフィール → 企業マッチング |
| **LAPRAS** | 公開アウトプットの自動収集 | スコアリング → 企業スカウト |
| **Smart Profile** | **テキスト・音声・画像の記録** | **AIプロフィール → マッチング → 名刺 → モチベーション可視化** |

Smart Profileだけが、記録データを**複数の対外的な価値**（プロフィール、マッチング、デジタル名刺、モチベーショングラフ）に変換できる。他のサービスはデータが個人の中で閉じるか、出口が単一（企業マッチングのみ）に限定されている。

### 優位性3：「価値観の変化」を可視化する — 競合が最も弱い領域

ユーザーから提供された競合分析が指摘する通り、現在の市場で最も手薄な領域は**「長期の記録から価値観の変化を抽出し、人生の岐点の意思決定に使う」**というポジションである。

Beeは日々のパターンを学習するが、「半年前の自分と今の自分がどう変わったか」を可視化する機能はない。Apple JournalやDay Oneは振り返りを促すが、AIが長期トレンドを分析して変化を示す機能はない。

Smart Profileのモチベーショングラフと、蓄積データに基づくAI分析は、まさにこの空白を埋めるものである。「過去の自分との比較」「成長の軌跡の可視化」「価値観の変遷の言語化」は、Smart Profileが独占できる差別化ポイントである。

### 優位性4：学生に完全特化 — 既存プレイヤーが手を出しにくい領域

| ライフサイクル段階 | 対応する競合 | Smart Profile |
|---|---|---|
| 日常の記録・振り返り | Bee, Omi, Apple Journal, Day One | ◎ 音声・テキスト・画像で記録 |
| 自己分析・強みの発見 | なし（手動ツールのみ） | ◎ AIが自動で分析・抽出 |
| 価値観の変化の可視化 | なし | ◎ モチベーショングラフ |
| プロフィール・ポートフォリオ | Wantedly（書類ベース）, LAPRAS（エンジニアのみ） | ◎ 記録から自動生成 |
| 友人・仲間探し | Penmark（同じ授業のみ）, Pairs（恋愛のみ） | ◎ 価値観ベースの多角的マッチング |
| 就活・キャリア形成 | Wantedly, LinkedIn | ◎ デジタル名刺 + 企業マッチング |

Bee、Omi、LimitlessなどのウェアラブルAI系サービスは一般消費者向けであり、学生特化の機能を持たない。Penmarkは学生特化だが履修管理に限定される。Wantedlyは学生向け機能を持つが、社会人中心の設計である。Smart Profileは**学生のライフサイクル全体を一貫してカバーする唯一のプラットフォーム**である。

### 6.2. 主要リスクと対策

| リスク | 内容 | 対策 |
|---|---|---|
| **Beeの学生向け展開** | Amazon買収後、教育市場への拡張の可能性 | 「プロフィール生成→マッチング」という出口の優位性を早期に確立する。Beeにはない「対外共有」の価値を訴求 |
| **Wantedlyの動的生成への進化** | 書類ベースから日常記録ベースへの進化の可能性 | 学生特化のUXと、友人・ロールモデルマッチングという独自カテゴリで差別化 |
| **プライバシー懸念** | 日常記録の蓄積に対するユーザーの抵抗感 | 「記録タイミングはユーザーが制御」する設計を明示。Apple Journalと同等のプライバシー保護を訴求 |
| **コールドスタート問題** | 記録が少ない初期段階でのAI精度の低さ | オンボーディングで基本情報を収集し、早期にプロフィールの骨格を生成。少量データでも価値ある分析を提供 |
| **機能過多による軸ブレ** | タスク管理等がプロフィール生成の軸を薄める | プロフィール生成を最上位に据え、他機能は控えめに配置（現在のUI方針を維持） |

## 7. 結論：Smart Profileが勝つための3つの戦略

競合分析の結果、Smart Profileが差別化すべきポイントは以下の3つに集約される。

**戦略1：「記録」ではなく「価値観の変化」を可視化する。** Bee・Omi・Plaudなどの競合は「記録すること」自体に価値を置いているが、Smart Profileは記録を手段とし、**「あなたがどう変わったか」を見せること**に価値を置くべきである。モチベーショングラフ、スキルの成長推移、興味関心の変遷など、「過去の自分との比較」を中核体験とする。

**戦略2：「就活・進路選択」という人生の岐点に特化する。** Beeは記憶補助、Otterは議事録、Apple Journalは内省と、各競合にはそれぞれの「使い場面」がある。Smart Profileの使い場面は**「就活の自己PR」「進路選択の判断材料」「初対面での自己紹介」**である。この具体的なユースケースに特化することで、汎用的なライフログツールとの差別化が明確になる。

**戦略3：データに「出口」を作り続ける。** 競合の最大の弱点は、蓄積したデータが個人の中で閉じることである。Smart Profileは「プロフィール→マッチング→デジタル名刺」という対外的な出口を持つことで、**記録する動機**をユーザーに与え続ける。「記録すればするほど、良い出会いが増える」という正のフィードバックループが、Smart Profileの最大の競争優位となる。
